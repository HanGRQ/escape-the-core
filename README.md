# Escape the Core

An adaptive game-based learning platform that teaches IBM SkillsBuild's *Introduction to Large Language Models* course through an escape-room narrative. Players work through three acts, guided by an AI tutor (Doctor K) whose support intensity adapts in real time based on performance.

## Setup

### 1. Clone and install

```bash
git clone https://github.com/your-username/escape-the-core.git
cd escape-the-core
```

### 2. Backend

```bash
cd backend
python -m venv venv
source venv/bin/activate        # Windows: venv\Scripts\activate
pip install -r requirements.txt
```

Copy `.env.example` to `.env` and fill in your values:

```bash
cp .env.example .env
```

```env
# backend/.env

ANTHROPIC_API_KEY=sk-ant-...

# Firebase service account (choose one method)
# Method A — path to downloaded service account JSON
GOOGLE_APPLICATION_CREDENTIALS=/path/to/serviceAccountKey.json

# Method B — individual environment variables
FIREBASE_PROJECT_ID=your-project-id
FIREBASE_PRIVATE_KEY_ID=...
FIREBASE_PRIVATE_KEY="-----BEGIN RSA PRIVATE KEY-----\n..."
FIREBASE_CLIENT_EMAIL=firebase-adminsdk-...@your-project.iam.gserviceaccount.com
FIREBASE_CLIENT_ID=...
```

Build the knowledge base (run once):

```bash
python scripts/build_knowledge_base.py
```

This vectorises `knowledge_base/chunks.json` into ChromaDB at `knowledge_base/chroma_db/`.

Start the backend:

```bash
uvicorn app.main:app --reload --port 8000
```

### 3. Frontend

```bash
cd frontend
npm install
```

Copy `.env.example` to `.env`:

```bash
cp .env.example .env
```

```env
# frontend/.env
# Get these from Firebase Console > Project settings > Your apps > Web app

VITE_FIREBASE_API_KEY=...
VITE_FIREBASE_AUTH_DOMAIN=your-project.firebaseapp.com
VITE_FIREBASE_PROJECT_ID=your-project-id
VITE_FIREBASE_APP_ID=...
```

Start the frontend:

```bash
npm run dev
```

Open `http://localhost:5173`.

## Firebase Setup

1. Go to [Firebase Console](https://console.firebase.google.com) and create a project.
2. **Authentication** → Sign-in method → Enable: Email/Password, Google, Anonymous.
3. **Firestore** → Create database (start in test mode for development).
4. **Project settings** → Service accounts → Generate new private key (for the backend).
5. **Project settings** → Your apps → Add a Web App (for the frontend `.env` values).

## How It Works

### Dynamic Difficulty Adjustment (DDA)

Every answer submission is evaluated against three signals:

| Signal | Threshold | State |
|--------|-----------|-------|
| Consecutive errors | ≥ 3 | `STUCK` |
| Consecutive errors | ≥ 2 | `STRUGGLING` |
| Response time > 2× personal average (min 3 prior attempts) | — | `CONFUSED` |
| Hint button clicked | — | `CONFUSED` |
| Otherwise | — | `FLOW` |

`STRUGGLING` and `STUCK` states trigger a scaffold in the UI. All state transitions and latency values are logged to Firestore when a room is completed.

### Dual-Track RAG

When a wrong answer is submitted, the backend retrieves support content via two parallel tracks:

- **Track A** — metadata filter: matches chunks by `game_room`, `dda_trigger`, `difficulty=basic`, and content type appropriate for the current DDA state.
- **Track B** — semantic search: embeds the player's wrong answer and finds the most semantically similar chunks in the same room.

Results are merged, de-duplicated, and passed to the Claude API to generate Doctor K's response.

### Doctor K Persona

Doctor K's tone advances through five stages as the player completes each act:

| After | Persona | Address |
|-------|---------|---------|
| Start | Cold | "Visitor" |
| Act I | Collaborative | "Collaborator" |
| Act II | Caring | "Collaborator" |
| Act III | Ally | "Partner" |
| Quiz pass | Full unlock | player's name |

## API Reference

| Method | Endpoint | Description |
|--------|----------|-------------|
| `POST` | `/api/session/start` | Create or resume a session |
| `GET` | `/api/room/{id}/teach` | SSE — stream Doctor K's lecture |
| `POST` | `/api/room/{id}/chat` | SSE — player Q&A with Doctor K |
| `POST` | `/api/room/{id}/submit` | Submit answer; returns DDA state + feedback |
| `GET` | `/api/room/{id}/hint` | Trigger help-seeking; returns RAG hint |
| `POST` | `/api/room/{id}/complete` | Mark room complete; advance persona |
| `POST` | `/api/prompt/evaluate` | Evaluate Act III prompt (5-dimension rubric) |
| `POST` | `/api/quiz/submit` | Submit final quiz; issue certificate if ≥ 75% |
| `GET` | `/api/progress/{user_id}` | Retrieve full player progress |
| `GET` | `/health` | Health check |
